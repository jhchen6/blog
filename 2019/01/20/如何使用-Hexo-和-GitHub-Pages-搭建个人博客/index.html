下面简单地介绍如何利用 GitHub Pages 来搭建个人博客。

前置要求

需要安装

  1. Node.js
  2. Git

如果你在中国大陆境内，推荐再安装 cnpm，并把下面所有命令中的 npm 换成 cnpm。cnpm 用了中国境内的淘宝镜像，安装各种包的速度会更快。

安装 cnpm：

在 GitHub 上建库

  • 如果想用 GitHub Pages 的个人主页（一个账号只能有一个），建库起名 <username>.github.io ；最后博客网址会是 <username>.github.io。
  • 如果想用项目主页（可以有无限多个），起什么名字都行；博客网址是 <username>.github.io/<reponame>。
  • 必须是 public 的库，不能设成 private。
  • 可以选上自动添加 README.md 的选项，方便下一步操作。

接下来到 Settings -> GitHub Pages -> Source，选择 master branch，Save。

如果前面没有选添加 README.md，此时 Repo 还完全是空的，这个 Source 选项就只能选 None。

也可以等到把本地内容推送上来之后再设置。

命令解释

以上所谓发布，其实是把你的源文件、素材等等之类的东西解析成可以直接部署的静态网页，然后推送到 GitHub 仓库里，然后 GitHub 会用它形成可以访问的网页。

生成的静态网页位于 public 目录下，只有这个文件夹里的内容会被推送到 GitHub 仓库，而且每次推送都会把之前的内容完全覆盖。

clean

这个命令用于删除之前生成内容，例如 public 文件夹。如果你确定新旧生成之间没什么冲突，也可以不删除，生成会快一些（大概吧）。

generate

生成静态网页，放到 public 文件夹。

deploy

根据我们的配置，这里 deploy 就是推送 public 文件夹到远端 GitHub 仓库。
